二叉树的定义

二叉树：每个结点最多有两棵子树

  • 左子树（Left Subtree）
  • 右子树（Right Subtree）

0 个子树 → 叶子结点

1 个子树 → 只有左孩子，或只有右孩子

2 个子树 → 左右孩子都有

重要："左右"有区别

只有左孩子的树 ≠ 只有右孩子的树

二叉树的基本性质
性质一

第 i 层最多有 2i-1 个结点

i=1: 1
i=2: 2
i=3: 4
i=4: 8
性质二

深度为 k 的二叉树最多有 2k - 1 个结点

等比数列求和：2⁰ + 2¹ + ... + 2k-1 = 2k - 1

k = 3 时，最多 7 个结点　|　k = 4 时，最多 15 个结点

性质三

对任意二叉树：n₀ = n₂ + 1

n₀ = 度 0　n₁ = 度 1　n₂ = 度 2

① 总边数 = n₁ + 2n₂（从度数看）

② 总边数 = n₀ + n₁ + n₂ - 1（从父子关系看）

③ 联立 ⟹ n₀ = n₂ + 1

示例验证

非常有用，考试面试常考！

满二叉树（Full Binary Tree）
定义

每一层结点数都达到最大值，所有叶子都在最底层

特点
  • 没有度为 1 的结点：n₁ = 0
  • 每个分支结点恰好有 2 个孩子
  • 叶子只出现在最后一层
  • 总结点数 = 2k - 1（k 为深度）
层序编号

从上到下、从左到右依次编号 1, 2, 3, ..., 2k-1

结点 i 的左孩子 = 2i

结点 i 的右孩子 = 2i + 1

结点 i 的父结点 = ⌊i / 2⌋

完全二叉树（Complete Binary Tree）

除最后一层外全满，最后一层叶子从左到右连续出现

关键：层序编号不能跳过任何位置

✓ 正例

✗ 反例

重要性：完全二叉树是数组存储的前提，也是堆的基础结构

二叉搜索树（BST）
定义
  • 左子树所有结点值 < 根值
  • 右子树所有结点值 > 根值
  • 左右子树本身也是 BST
核心性质

中序遍历 BST

得到严格递增的有序序列

中序遍历

左子树 → 根 → 右子树

按这个顺序访问 BST 的每个结点，结果就是从小到大排好序的

二叉树的存储结构
数组存储

适合完全二叉树 / 堆

层序遍历顺序存入数组，根在索引 0

结点 i 的左孩子 = 2i + 1

结点 i 的右孩子 = 2i + 2

结点 i 的父结点 = (i - 1) // 2

# 数组表示的二叉树（完全二叉树） tree = [1, 2, 3, 4, 5, 6] def left(i): return 2 * i + 1 def right(i): return 2 * i + 2 def parent(i): return (i - 1) // 2 print(f"根: {tree[0]}") print(f"结点 2 的左孩子: {tree[left(1)]}") print(f"结点 5 的父结点: {tree[parent(4)]}")

优点

无指针、O(1) 计算父子

缺点

普通二叉树空间浪费大

链表存储

最通用

每个结点包含值 + 左指针 + 右指针

class TreeNode: def __init__(self, val=0): self.val = val self.left = None self.right = None

优点

灵活、无空间浪费

缺点

需要指针空间、不能下标定位
