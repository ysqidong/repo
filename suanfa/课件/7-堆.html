上浮 Shift Up

点击「下一步」开始上浮
def shift_up(arr, i): while i > 0: p = (i - 1) // 2 if arr[p] >= arr[i]: break arr[p], arr[i] = arr[i], arr[p] i = p
下沉 Shift Down
数组：[20, 15, 18, 10, 12, 16, 14] → 删除 20
点击「下一步」开始下沉
def shift_down(arr, n, i): while 2*i + 1 < n: l = 2*i + 1 r = 2*i + 2 mx = l if r < n and arr[r] > arr[l]: mx = r if arr[mx] <= arr[i]: break arr[mx], arr[i] = arr[i], arr[mx] i = mx

为什么 Floyd 建堆是 O(n)？
节点位置数量最多下沉层数总工作量
叶子节点n/200
倒数第 2 层n/41n/4
倒数第 3 层n/822n/8 = n/4
倒数第 4 层n/1633n/16
…………
根节点1log nlog n
大量节点在底部，下沉层数少；少量节点在顶部，下沉层数多。"数量多"和"下沉深"成反比，总工作量收敛于 O(n)
粗略证明：高度 h 的结点最多 n/2h+1 个，每个下沉最多 h 层。总工作量 = Σ (n/2h+1)·h = n·Σ h/2h+1，级数收敛于 O(n)
TopK 问题 ⭐⭐

从 n 个数中找出 最大的 K 个

方法时间复杂度空间复杂度
全局排序O(n log n)O(1) 或 O(n)
大顶堆弹 K 次O(n + K log n)O(n)
小顶堆法 ⭐O(n log K)O(K)
小顶堆法详解
  1. 用前 K 个元素建小顶堆（堆顶是当前 TopK 中最小的）
  2. 遍历剩余 n-K 个元素：若 x > 堆顶，则替换堆顶并下沉
  3. 最终堆内即 TopK
def topk(nums, k): # Python heapq 默认小顶堆 import heapq heap = nums[:k] heapq.heapify(heap) # O(k) for x in nums[k:]: if x > heap[0]: heapq.heapreplace(heap, x) # O(log k) return heap

时间 = O(k) + (n-k)·O(log k) = O(n log k)　|　空间 = O(k)

应用场景
  • OS 进程调度：高优先级任务先执行
  • Dijkstra 最短路径：每次取距离最小的结点
  • 合并 K 个有序链表：K 路归并，每次取最小头结点
  • 任务调度 / 定时器：最近到期的任务先触发
附录：Python 协议类 Protocol
typing.Protocol · 结构子类型 · 鸭子类型的正规化
什么是 Protocol？
  • PEP 544 引入，Python 3.8+ 标准库提供
  • 属于结构性子类型（Structural Subtyping）：不检查继承关系，只检查“有没有对应的方法/属性”
  • 相当于把 Python 的鸭子类型用静态类型系统“正规化”
Protocol vs ABC 对比
维度ABC（抽象基类）Protocol（协议类）
继承方式必须显式继承 class C(A)无需继承，自动识别
检查时机运行时 isinstance()静态类型检查器（mypy）
子类型判定名义子类型（Nominal）结构子类型（Structural）
性能影响运行时有一定开销零运行时开销
类比“必须认爹”“长得像就行”
代码示例
from typing import Protocol # 1. 定义协议 class PushPop(Protocol): def push(self, x: int) -> None: ... def pop(self) -> int: ... # 2. 实现类（不继承 PushPop！） class MyHeap: def push(self, x: int) -> None: ... def pop(self) -> int: ... # 3. 静态类型检查通过 def use(q: PushPop) -> None: q.push(1) h = MyHeap() use(h) # ✔ mypy 认可，运行时无感知
为什么优先队列用 Protocol 更好？
  • 零侵入：已有类无需改继承链即可充当优先队列
  • 解耦：ADT 定义与实现彻底分离
  • 天然符合：任何带 push/pop 的类都自动满足
一句话总结：ABC 是“必须认爹”，Protocol 是“长得像就行”。
