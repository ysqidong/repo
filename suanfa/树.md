

# Python 树（Tree）数据结构学习笔记

树是一种非常核心的非线性数据结构。无论是在配置服务器时面对的 Linux 文件目录层级，还是在编写 RPG 游戏时管理复杂的游戏场景与对象嵌套关系，其底层逻辑都高度依赖树形结构。

### 一、 核心概念与术语

在深入代码之前，我们需要明确树的几个基础解剖学概念：

* **节点 (Node)：** 树的基本构成单位，包含数据和指向子节点的引用。
* **根节点 (Root)：** 树的顶端节点，一棵树只有一个根节点（类似于 Linux 的 `/` 目录）。
* **边 (Edge)：** 连接两个节点的线。
* **父节点 / 子节点 (Parent / Child)：** 上下级关系的相对称呼。
* **叶子节点 (Leaf)：** 没有子节点的末端节点。
* **深度 (Depth) / 高度 (Height)：** 深度是指从根节点到该节点的最长路径；高度是指从该节点到最远叶子节点的最长路径。

### 二、 Python 中的通用树实现

Python 标准库中并没有内置的树形数据结构，但我们可以利用类（Class）和列表（List）非常直观地构建一棵通用树。

#### 1. 通用树节点类的定义

```python
class TreeNode:
    def __init__(self, data):
        self.data = data
        self.children = []  # 使用列表存储多个子节点
        self.parent = None  # 可选：记录父节点

    def add_child(self, child_node):
        child_node.parent = self
        self.children.append(child_node)

    def __repr__(self):
        return f"TreeNode({self.data})"

```

#### 2. 构建通用树实例

```python
# 构建一棵简单的系统菜单树
root = TreeNode("Main Menu")
setting = TreeNode("Settings")
gameplay = TreeNode("Gameplay")

root.add_child(setting)
root.add_child(gameplay)

audio = TreeNode("Audio")
video = TreeNode("Video")
setting.add_child(audio)
setting.add_child(video)

```

### 三、 二叉树 (Binary Tree)

在实际算法和数据分析中，**二叉树**是最常被使用的变体。它的严格限制是：**每个节点最多只能有两个子节点**（分别称为左子节点和右子节点）。

#### 1. 二叉树的节点定义

```python
class BinaryTreeNode:
    def __init__(self, value):
        self.value = value
        self.left = None   # 左子节点
        self.right = None  # 右子节点

```

#### 2. 树的遍历算法 (Tree Traversal)

遍历是指按照特定规则访问树中的每一个节点且仅访问一次。由于二叉树不是线性结构，遍历主要分为两大类：**深度优先搜索 (DFS)** 和 **广度优先搜索 (BFS)**。

##### (1) 深度优先遍历：中序遍历 (In-order)

常用于二叉搜索树，遍历会得到一个有序序列。顺序为：左子树 -> 根节点 -> 右子树。

```python
def inorder_traversal(node):
    if node is not None:
        inorder_traversal(node.left)
        print(node.value, end=" -> ")
        inorder_traversal(node.right)

```

##### (2) 深度优先遍历：前序遍历 (Pre-order)

顺序为：根节点 -> 左子树 -> 右子树。

```python
def preorder_traversal(node):
    if node is not None:
        print(node.value, end=" -> ")
        preorder_traversal(node.left)
        preorder_traversal(node.right)

```

##### (3) 深度优先遍历：后序遍历 (Post-order)

常用于安全删除整棵树或计算目录大小。顺序为：左子树 -> 右子树 -> 根节点。

```python
def postorder_traversal(node):
    if node is not None:
        postorder_traversal(node.left)
        postorder_traversal(node.right)
        print(node.value, end=" -> ")

```

##### (4) 广度优先遍历：层序遍历 (Level-order)

按树的层级从上到下、从左到右逐层访问，通常借助队列（Queue）来实现。

```python
from collections import deque

def level_order_traversal(root):
    if not root:
        return
    
    queue = deque([root])
    while queue:
        current = queue.popleft()
        print(current.value, end=" ")
        
        if current.left:
            queue.append(current.left)
        if current.right:
            queue.append(current.right)

```

一、 二叉树的核心数学性质设二叉树的层数为 $k$（根节点在第 1 层），节点总数为 $N$，叶子节点数为 $n_0$，度为 2 的节点数为 $n_2$。第 $i$ 层的最大节点数：在二叉树的第 $i$ 层上，最多有 $2^{i-1}$ 个节点（$i \ge 1$）。深度为 $k$ 的二叉树最大节点数：一棵深度为 $k$ 的二叉树，最多有 $2^k - 1$ 个节点（此时为满二叉树）。度数关系定理（极其重要）：对任何一棵二叉树，如果其叶子节点数为 $n_0$，度为 2 的节点数为 $n_2$，则必定满足：$n_0 = n_2 + 1$简易逻辑推导：总边数 $E = N - 1$（除了根节点，每个节点头顶都有一条边）；同时从向下发射的角度看，$E = n_1 + 2n_2$。由此可得 $n_0 + n_1 + n_2 - 1 = n_1 + 2n_2$，化简即得 $n_0 = n_2 + 1$。完全二叉树的深度：具有 $N$ 个节点的完全二叉树的深度为 $\lfloor\log_2 N\rfloor + 1$。
二、 深度优先遍历（DFS）的应用场景与工程条件深度优先遍历根据访问根节点（V）与左右子树（L、R）的相对顺序，分为三种。在具体的系统开发（如游戏树、编译原理语法树）中，它们的工程应用条件有着严格的区分：1. 前序遍历 (Pre-order, VLR)算法顺序：根节点 $\rightarrow$ 左子树 $\rightarrow$ 右子树。应用条件与场景：树的复制/序列化：当需要将内存中的树形结构（如物理引擎中的碰撞体层级）完整保存到硬盘，或者传输给网络另一端时，前序遍历能够首先确定根节点，是最天然的序列化手段。目录结构生成：在配置 Linux 文件系统或渲染 RPG 游戏菜单时，需要优先打印出父目录/主菜单，再递归打印子目录。在线演示工具：你可以通过 Visualgo - Binary Search Tree 或 Data Structure Visualizations - BT 交互式地观察前序遍历的逻辑。
2. 中序遍历 (In-order, LVR)算法顺序：左子树 $\rightarrow$ 根节点 $\rightarrow$ 右子树。应用条件与场景：二叉搜索树（BST）的有序输出：二叉搜索树满足“左子树 < 根节点 < 右子树”的底层拓扑关系。因此，对其进行中序遍历，可以在线性时间复杂度 $O(N)$ 内得到一个严格升序的数组。表达式树的求值：在解析数学公式（如 A + B）的语法树时，中序遍历能够最直观地还原符合人类逻辑的算术表达式。
3. 后序遍历 (Post-order, LRV)算法顺序：左子树 $\rightarrow$ 右子树 $\rightarrow$ 根节点。应用条件与场景：依赖性计算（计算目录大小/耗能）：你必须先知道左子目录和右子目录的大小，才能相加算出父目录的总大小。内存安全释放（析构函数）：在底层 C++ 或手动管理内存的 Python 脚本中，销毁一棵树时，必须先释放左、右子节点的内存，最后才能安全地删除根节点。如果先删根节点，子节点的指针就会丢失，造成严重的内存泄漏。
4. 三、 广度优先遍历（BFS/层序遍历）的应用场景层序遍历 (Level-order)算法顺序：按树的深度，从上到下、从左到右逐层扫描。应用条件与场景：寻找最短路径/最少步骤：在博弈树（如棋类 AI、游戏寻路）中，层序遍历代表优先搜索距离当前状态最近的可能，适合解决“最少走几步能通关”的问题。树的水平剪枝：当需要限制游戏渲染树的深度、或者限制服务器配置文件读取的层级时，使用层序遍历可以在达到指定深度时立即终止循环。
