### 一句话说清 Docker 是什么

**Docker 就像一个“免安装、直接点开即用”的 App 包。**

以前你装一个软件（比如 MySQL 数据库），需要：下载安装包 -> 解压 -> 配置环境 -> 运行。麻烦且容易出错。

现在用 Docker，你只需要一条命令，它就会自动把 MySQL 和它需要的所有环境打包下载好，然后瞬间启动。**不用手动安装，不用配置环境。**

### 三个核心概念（用做饭类比）

1.  **镜像**：**菜谱**。只记录做法和配料，不能吃。比如“鱼香肉丝菜谱”。
2.  **容器**：**做好的菜**。按照菜谱做出来的实物，可以吃（运行）。你可以同时炒很多盘鱼香肉丝。
3.  **仓库**：**网上菜谱大全**。里面存了无数别人做好的菜谱（镜像），你直接下载就行。

所以 Docker 的工作流就是：**从网上菜谱（仓库）下载菜谱（镜像） -> 照着菜谱做成菜（容器）**。

---

### 你最常用的 3 条命令

打开终端（命令行），记住这三条就够了：

| 你想做什么 | 命令 | 解释 |
| :--- | :--- | :--- |
| **下载一个软件** | `docker pull mysql` | 从网上下载 MySQL 的镜像（菜谱） |
| **运行这个软件** | `docker run -d -p 3306:3306 mysql` | 用菜谱做一道菜（启动容器），并且把电脑的3306端口指向它 |
| **查看正在运行的软件** | `docker ps` | 看看哪道菜（容器）正在做 |

**实际例子：** 想装一个 Nginx 网页服务器？

不需要去官网下载、配置。只需要打开终端，输入：

```bash
docker run -d -p 8080:80 nginx
```

然后浏览器访问 `http://localhost:8080`，你就看到 Nginx 的欢迎页了。整个过程不到 10 秒，而且你的电脑干干净净，没有任何残留。

### Docker 和虚拟机最直观的区别

-   **虚拟机**：像在你电脑里**装了一个完整的 Windows 系统**。很重，启动要几分钟，占用好几个 G 的空间。
-   **Docker**：像在你电脑里**打开一个微信小程序**。很轻，启动只要 1 秒，占用几十 M 的空间。

**记住：虚拟机虚拟的是“完整的电脑”，Docker 虚拟的只是“一个进程（一个软件）”。**

---

### 怎么删除不需要的东西？

| 你想做的事 | 命令 |
| :--- | :--- |
| 停止正在运行的软件 | `docker stop 容器ID`（ID 用 `docker ps` 查看） |
| 删除这个软件（删除容器） | `docker rm 容器ID` |
| 删除下载的菜谱（删除镜像） | `docker rmi 镜像名` |
| **一键大扫除**（删除所有停止的容器、没用的镜像） | `docker system prune` |

### 总结一张图

```
你之前装软件：  去官网下载 -> 双击安装 -> 下一步下一步 -> 配置环境变量 -> 运行
用 Docker 装：  打开终端 -> 输入 docker run ... -> 运行完成
```

**你不需要记复杂的命令**，平时最常用的就是 `docker pull`（下载）和 `docker run`（运行）。
