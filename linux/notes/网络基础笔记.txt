# 网络基础笔记

## DNS（域名系统）
- **作用**：把域名转换成 IP 地址。
- **简单理解**：域名是人记住的名字，IP 是机器识别的地址。
- **常见记录**：
  - A：域名 -> IPv4 地址
  - AAAA：域名 -> IPv6 地址
  - CNAME：域名别名 -> 另一个域名
  - MX：邮件服务器记录
  - NS：负责解析的 DNS 服务器
  - TXT：存放验证信息，如 SPF、DKIM
  - SOA：域名区域的起始授权记录

## IPv4 和 IPv6
- **IPv4**：4 段数字，用点分隔，例如 192.168.1.1。
  - 地址数量约 42 亿个，已经不够用了。
- **IPv6**：8 段 16 进制数，用冒号分隔，例如 2001:0db8:85a3:0000:0000:8a2e:0370:7334。
  - 地址数量非常大，解决了 IPv4 地址不足的问题。

## 端口
- **定义**：端口是“网络服务的房间号”。
- **关系**：IP 地址表示设备，端口表示设备上的某个程序或服务。
- **例子**：192.168.1.10:80 表示这个设备上的网页服务。
- **常见端口**：
  - 80：HTTP
  - 443：HTTPS
  - 22：SSH
  - 53：DNS
  - 3306：MySQL
- **端口范围**：0-65535。
  - 0-1023：系统端口
  - 1024-49151：注册端口
  - 49152-65535：私有/动态端口

## 协议
- **定义**：协议就是“通信规则”。
- **作用**：规定两台设备之间怎么说话、怎么发送和接收数据。
- **类比**：人和人要说同一种语言才能交流，电脑之间要按同一种协议才能通信。
- **常见网络协议**：
  - **TCP**：传输控制协议，可靠、按顺序传输数据，常用于网页、文件传输、邮件。
  - **UDP**：用户数据报协议，速度快、不保证顺序、不保证重传，常用于视频、语音、DNS 查询。
  - **HTTP**：超文本传输协议，浏览器访问网页时用的协议。
  - **HTTPS**：安全版 HTTP，加密后的网页访问协议。
  - **DNS**：域名解析协议，把域名转换成 IP。
  - **SSH**：安全远程登录协议。
- **协议、IP、端口关系**：
  - IP：找设备
  - 端口：找程序
  - 协议：说话的规则
  - **例子**：
    - TCP + 80：常用于网页服务
    - UDP + 53：常用于 DNS 查询

    SSL/TLS 是保障互联网通信安全的加密协议。浏览器地址栏的 🔒小锁图标 就是它在工作的标志。

2. 历史与关系
SSL：老协议，已过时且不安全。

TLS：SSL 的继任者，当前真正在用的标准。

习惯叫法：常说的"SSL证书"实际应叫"TLS证书"。

主流版本：TLS 1.2 和 TLS 1.3（推荐1.3，更快更安全）。

3. 工作原理（4步握手）
打招呼+发证书：服务器向浏览器出示CA颁发的"身份证"（数字证书）。

验证身份：浏览器检查证书是否真实有效。

协商会话密钥：双方安全地生成一个临时专用密钥（有"前向保密"特性）。

加密通信：之后所有数据都用此密钥加密传输。

前向保密：即使网站的私钥未来泄露，过去的聊天记录也无法被解密。
