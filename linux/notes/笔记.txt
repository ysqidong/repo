tmux（Terminal Multiplexer）是一个终端多路复用器，允许你在单个终端窗口中创建和管理多个会话、窗口和面板。它特别适合远程工作，因为会话可以持久化，即使断开连接也不会丢失。
基本使用：

启动：tmux
新建窗口：Ctrl-b c
切换窗口：Ctrl-b n（下一个）或 Ctrl-b p（上一个）
分割面板：Ctrl-b %（垂直）或 Ctrl-b "（水平）
切换面板：Ctrl-b 方向键
分离会话：Ctrl-b d（后台运行）
重新连接：tmux attach 或 tmux a
常用命令：

列出会话：tmux ls
杀死会话：tmux kill-session -t 会话名
配置：编辑 ~/.tmux.conf 添加自定义键绑定。

crontab定时任务
crontab -e  # 编辑当前用户的 crontab 文件
crontab -l  # 查看当前用户的 crontab 任务
crontab -r  # 删除当前用户的 crontab 任务
Crontab 是 Linux/Unix 系统中的定时任务调度工具，用于在指定时间自动执行命令或脚本。以下是基本用法：

基本语法
时间格式
每行任务格式：分钟 小时 日 月 周 命令

分钟：0-59
小时：0-23
日：1-31
月：1-12
周：0-7（0 和 7 都表示周日）
示例：
# 每天凌晨 1:00 备份文件
0 1 * * * /usr/bin/tar -czf /backup/backup.tar.gz /home/user/data

# 每周一早上 8:00 发送邮件
0 8 * * 1 echo "Weekly report" | mail -s "Report" user@example.com

* * * * * command：每分钟执行
0 2 * * * command：每天凌晨 2:00 执行
*/5 * * * * command：每 5 分钟执行
0 0 1 * * command：每月 1 日凌晨执行

tar归档+压缩
归档（不压缩）：tar -cf archive.tar file1 file2 dir/
-c：创建归档
-f：指定文件名
压缩归档：
gzip：tar -czf archive.tar.gz file1 file2 dir/（常用，扩展名 .tar.gz）

xz：tar -cJf archive.tar.xz file1 file2 dir/（.tar.xz）

-x：提取
-z：gzip，gunzip

DNS 这是啥？
DNS 就是“域名翻译器”。

电脑只认 IP 地址，比如 192.168.1.1
人喜欢记域名，比如 example.com
DNS 负责把域名翻成 IP
就像：

你记名字
通讯录帮你找到电话号码
常见 DNS 记录类型（最容易理解）
A

把域名翻成 IPv4 地址
例子：example.com -> 192.0.2.1
AAAA

把域名翻成 IPv6 地址
例子：example.com -> 2001:db8::1
CNAME

一个域名当做别名，指向另一个域名
例子：www.example.com -> example.com
MX

邮件用的
告诉别人：给这个域名发邮件，哪个邮箱服务器收
例子：example.com -> mail.example.com
NS

告诉别人：谁是这个域名的“负责 DNS 的服务器”
例子：example.com -> ns1.example.com
TXT

放一些文字信息
常用于验证、邮箱安全设置（SPF、DKIM）
例子：example.com -> "v=spf1 include:..."
SOA

域名区域的“配置说明书”
记录谁是主服务器、管理员邮箱、更新频率等
每个域名一定要有一个
最简单理解
A / AAAA：域名直接对应一个地址
CNAME：域名的“别名”
MX：邮件去哪里送
NS：谁管这个域名的解析
TXT：放一些验证信息
SOA：域名配置说明书

IPv4
全称：Internet Protocol version 4
形式：4 个十进制数，每个 0-255，用点分隔
例子：192.168.1.1
最大地址数量约 42 亿个
现在已经快不够用了
IPv6
全称：Internet Protocol version 6
形式：8 组 16 进制数，用冒号分隔
例子：2001:0db8:85a3:0000:0000:8a2e:0370:7334
地址数量非常大，几乎不会用完
主要是为了解决 IPv4 地址不够的问题
