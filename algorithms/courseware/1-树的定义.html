📐 核心术语（一）

根 Root

树的最顶层结点，一棵树有且只有一个根

结点 Node

树中的每一个元素都叫结点

边 Edge

连接两个结点的线，表示父子关系

叶子结点 Leaf

度为 0 的结点，即没有子结点的结点

类比：组织架构里的"基层员工"

📊 核心术语 — 度（Degree）

定义：一个结点拥有的 子树个数（即 孩子的数量）

A 的度 = 3（孩子 B、C、D）

B 的度 = 2（孩子 E、F）

C 的度 = 0（叶子）

E 的度 = 1（孩子 H）

树的度 = 所有结点中度数的最大值

此树的度 = 3（结点 A）

📏 核心术语 — 深度 / 层（Depth）

定义：从 根 到该结点的路径上的 边数

📌 约定：根深度 = 1

A 的深度 = 1

B / C / D 的深度 = 2

E / F / G 的深度 = 3

H 的深度 = 4

🏔️ 核心术语 — 高度（Height）

定义：从该结点到 最远叶子 的路径上的 边数

📌 约定：叶子高度 = 1

H（叶子）高度 = 1

E 的高度 = 2（E→H）

B 的高度 = 3（B→E→H 最长）

A 的高度 = 4（A→B→E→H 最长）

🔗 路径、祖先、子孙

路径 Path

从一个结点到另一个结点经过的结点序列

路径长度 = 边的数量

A → H 的路径：A→B→E→H

路径长度 = 3

祖先 Ancestor

从根到该结点路径上的所有结点

（包括根，不包括自己）

H 的祖先：A, B, E

子孙 Descendant

该结点下面所有能到达的结点

B 的子孙：E, F, H

父 / 子 结点

直接相连的一对结点

A 是 B 的 父，B 是 A 的 子

兄弟结点 Sibling

同一个父结点的孩子

B, C, D 互为兄弟

🔀 有序树 vs 无序树

有序树 Ordered Tree

  • 子树有 左右顺序 之分
  • 交换位置 → 得到 不同的树
  • 典型例子：二叉树

无序树 Unordered Tree

  • 子树 没有顺序
  • 交换位置 → 仍是同一棵树
  • 典型例子：家族族谱
💡 绝大多数算法问题中讨论的是 有序树
