Binary Search Tree
左 < 根 < 右 ｜ 查找 · 插入 · 删除
二叉搜索树的定义与性质
定义
  • 左子树所有结点值 < 根值
  • 右子树所有结点值 > 根值
  • 左右子树本身也是 BST
核心性质

中序遍历 BST

得到严格递增的有序序列

注意：是所有左子树结点都小于根，不只是直接左孩子！
本课统一示例树
中序: 20 → 30 → 40 → 50 → 60 → 70 → 80
查找操作 Search
查找逻辑
  • 从根开始比较
  • 目标值 < 当前结点 → 往左走
  • 目标值 > 当前结点 → 往右走
  • 相等 → 找到
  • 走到空 → 不存在
本质：二分搜索思想在树结构上的延伸

查找 40 的路径：

50 → 30 → 40

比较 3 次找到

def search(root, target): if not root: return None if target == root.val: return root if target < root.val: return search(root.left, target) return search(root.right, target)

插入操作 Insert
插入逻辑
  • 从根开始，按查找规则往下走
  • 走到空位置，就是插入点
  • 新结点总是作为叶子插入
关键：插入不会破坏 BST 性质，因为新结点总是挂在正确的一侧

插入 65 的路径：

50 → 70 → 60 → (右空位)

def insert(root, val): if not root: return TreeNode(val) if val < root.val: root.left = insert(root.left, val) elif val > root.val: root.right = insert(root.right, val) return root

删除 — 叶子结点
场景

待删除结点是叶子（没有孩子）

操作
  • 直接删除该结点
  • 父结点的对应指针置空
示例

删除叶子 20

30 的左指针由指向 20 变为 None

最简单的删除情况

时间复杂度：O(log n) ~ O(n)

删除 — 只有一个孩子
场景

待删除结点只有一个孩子（左或右）

操作
  • 用孩子顶替被删除结点的位置
  • 父结点直接指向孩子，跳过被删结点
示例

删除 70（只有一个左孩子 60）

50 的右指针由指向 70 变为指向 60

形象理解：就像链表删除中间结点，让孩子“继承”位置

删除 — 有两个孩子

最复杂的情况：用中序前驱或中序后继替换被删结点

中序前驱 Predecessor

定义：左子树的最右结点

即左子树中值最大的结点

中序后继 Successor

定义：右子树的最左结点

即右子树中值最小的结点

替换策略：用前驱/后继的值覆盖被删结点，然后递归删除那个前驱/后继（它最多只有一个孩子）

原始 BST 中删除根结点 50，两种策略的结果：

策略一：中序前驱 40

40 替换 50，再删除原 40

策略二：中序后继 60

60 替换 50，再删除原 60

两种策略都正确！ 选择任意一种即可，保证中序遍历结果仍然有序
